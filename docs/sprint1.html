ConwayLife Sprint1

Introduction

Realizzazione in Java del GAME OF LIFE DI CONWAY , limitatamente al requisito R1.

Requirements

  • R1: Sviluppare in Java il gioco Life di Conway come zero-player game. La griglia di celle evolve secondo le regole del gioco.
  • R2: L'utente deve poter:
    • definire la configurazione iniziale della griglia
    • osservare l'evoluzione del gioco ( vista del gioco )
    • interrompere e riprendere l'evoluzione
    • resettare la griglia quando il gioco è fermo

Requirement analysis

Cosa si intende per "Cella"?

public interface ICell {
    void setStatus(boolean alive);   // Imposta lo stato della cella
    boolean isAlive();               // Restituisce true se la cella è viva
    void switchCellState();          // Inverte lo stato della cella
}
Cosa si intende per "Griglia"?

public interface IGrid {
    int getRows();                              // Numero di righe
    int getCols();                              // Numero di colonne
    ICell getCell(int row, int col);           // Restituisce la cella specifica
    boolean isCellAlive(int row, int col);     // Stato della cella
    void setCellState(int row, int col, boolean alive); // Imposta lo stato della cella
    int countNeighbors(int row, int col);      // Numero di celle vicine vive
    void reset();                               // Reset della griglia
}
Cosa si intende per "Life"?

public interface LifeInterface {
    void nextGeneration();                         // Calcola la prossima generazione
    boolean isCellAlive(int row, int col);        // Controlla se la cella è viva
    void setCellState(int row, int col, boolean alive); // Imposta lo stato della cella
    ICell getCell(int row, int col);              // Restituisce la cella
    IGrid getGrid();                               // Restituisce la griglia
    void resetGrids();                             // Resetta la griglia
}
